# План розробки (MVP)

## Мета

Зробити безкоштовну україномовну веб-платформу з **100-денним** курсом тренувань “як Сотка” (за логікою й структурою, але з нашим контентом): реєстрація → програма → “сьогоднішнє тренування” → відмітка виконання → журнал прогресу.

## Принципи MVP

- **Web-first, API-first**: проєктуємо так, щоб Android потім підключився до того ж API.
- **Статична програма**: без “розумної” адаптації на старті.
- **Контент з нуля українською**: тексти + (опційно) відео після MVP.
- **Безкоштовно**: без платежів і paywall’ів.
- **Обладнання**: **турнік обов’язковий** (бруси — опційно).

## Нефункціональні вимоги

- **Безпека**: хешування паролів, захист сесій, базові rate limits (мінімально).
- **Продуктивність**: швидке завантаження тренування й журналу.
- **Зручність**: мобільна версія (основний сценарій — телефон).

## Основні модулі

- **Auth**: реєстрація/вхід/вихід.
- **Onboarding**: коротка анкета (рівень, обладнання, обмеження) + вибір програми.
- **Program**: програма → тижні → дні → вправи.
- **Workout**: сторінка дня (план) + відмітка виконання + нотатки.
- **Logs**: журнал (список/календар) + прогрес по тижнях.
- **Library**: бібліотека вправ (опис техніки).
- **Admin**: керування вправами й програмами.

## Етапи

### Етап A — Специфікація

- Зафіксувати структуру першої програми (**100 днів**, ритм календаря, список вправ, фази).
- Зафіксувати UX екрани та user flows.
- Зафіксувати модель даних та API-контракти.

### Етап B — Каркас застосунку

- База: Next.js, UI-стиль, навігація.
- Postgres + ORM.
- Auth і базові ролі (user/admin).

### Етап C — Програма та контент

- Створити адмінські CRUD для вправ/програм/днів.
- Заповнити контент українською (мінімально достатній для 1 програми).

### Етап D — Тренування та журнал

- Сторінка “сьогодні” + виконання дня.
- Журнал/історія, простий прогрес.

### Етап E — Полірування та реліз

- SEO для лендінгу, сторінки політик.
- Базова аналітика подій (опційно).
- Перевірка UX на телефоні.

## Definition of Done (MVP)

- Користувач може зареєструватися, пройти онбординг, отримати програму.
- На кожен день програми є сторінка з вправами українською.
- Користувач може відмічати виконання і бачити журнал.
- Адмін може керувати вправами й програмою.
