# MVP специфікація (веб)

## 1) Ролі

- **User**: проходить програму, читає контент, веде журнал.
- **Admin**: керує вправами, програмами, днями.

## 2) Основні сутності (концептуально)

- **Program**: курс (назва, опис, **100 днів**, ритм календаря/шаблон).
- **ProgramDay**: день програми (номер дня/тижня, заголовок).
- **Exercise**: вправа (опис українською).
- **DayExercise**: вправа в конкретному дні (схема підходи/повтори/відпочинок/темп).
- **WorkoutLog**: факт виконання користувачем дня (статус, нотатки, дата).
- **UserProfile**: мінімальні дані онбордингу (рівень, обладнання, обмеження).

## 3) Екрани та функції

### 3.1 Лендінг (`/`)

- Опис продукту, як проходити, FAQ, CTA “Почати”.

### 3.2 Auth

- **Реєстрація** (`/auth/register`): email, пароль.
- **Вхід** (`/auth/login`): email, пароль.
- **Відновлення пароля** (`/auth/forgot`, `/auth/reset`) — якщо встигаємо на MVP; інакше в backlog.

### 3.3 Онбординг (`/onboarding`)

- Поля:
  - рівень (самооцінка)
  - обладнання (**турнік обов’язковий**; бруси — опційно)
  - обмеження (плечі/коліна/спина + текстове поле)
  - вибір програми (на MVP — 1 програма за замовчуванням, або ручний вибір)

### 3.4 Дашборд (`/app`)

- “Сьогоднішнє тренування” (посилання на день).
- Прогрес по тижнях (виконано/заплановано).
- Швидкий доступ до журналу та бібліотеки вправ.

### 3.5 Сторінка дня (`/app/day/[dayId]`)

- Список вправ (картки):
  - назва
  - схема (підходи/повтори/відпочинок)
  - коротка підказка техніки
  - посилання “детальніше” на вправу
- Кнопка:
  - “Позначити виконаним” (з можливістю додати нотатку)

### 3.6 Журнал (`/app/log`)

- Список днів з датами та статусом (виконано/пропущено/заплановано).
- Фільтр по статусу (мінімально).

### 3.7 Бібліотека вправ (`/app/exercises`)

- Список + пошук.
- Картка вправи (`/app/exercises/[exerciseId]`): опис техніки, помилки, полегшення.

### 3.8 Адмінка (`/admin`)

- CRUD:
  - вправи
  - програми
  - дні програми
  - вправи в дні (порядок + схема)

## 4) API (попередній перелік)

> Якщо робимо Next.js API routes/server actions — це все одно формалізуємо як “контракти” для майбутнього Android.

### Auth

- `POST /api/auth/register`
- `POST /api/auth/login`
- `POST /api/auth/logout`
- (опційно) `POST /api/auth/forgot`
- (опційно) `POST /api/auth/reset`

### Program / Days

- `GET /api/me` (профіль + активна програма)
- `GET /api/programs` (список)
- `GET /api/programs/:id` (деталі)
- `GET /api/days/:id` (структура дня)
- `GET /api/today` (dayId “сьогодні”)

### Logs

- `POST /api/logs` (позначити день виконаним/пропущеним)
- `GET /api/logs` (історія)

### Admin

- `POST/PUT/DELETE /api/admin/`* (CRUD)
